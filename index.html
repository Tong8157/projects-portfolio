Honda Bass Max

Senior Capstone — Ohio State ECE · Honda Collaboration · Team of 5 · 2024–2025

A portable, in-vehicle acoustic testing system that localizes vibration-induced noise inside car cabins. The system uses four calibrated omnidirectional USB microphones, a Raspberry Pi 5, and a custom touchscreen GUI to capture audio from each seating position, subtract a known reference track via time-domain analysis, and generate spatial heatmaps pinpointing rattle and buzz sources — replacing tools costing tens of thousands of dollars with a sub-$1,000 solution.

Technology

  • Raspberry Pi 5
  • Python (Tkinter, NumPy, SciPy)
  • miniDSP UMIK-1 Microphones
  • STFT & RBF Interpolation
  • 7" LCD Touchscreen

Key Features

  • Real-time multi-mic recording
  • Time-domain noise subtraction
  • Spatial heatmap visualization
  • Per-seat audio playback
  • Bluetooth vehicle connectivity

Results

  • Tested in 4+ vehicle models
  • Setup in under 5 minutes
  • 20Hz–20kHz detection range
  • Under $1,000 total cost
  • Standalone portable operation

My Contributions

Led GUI development using Python/Tkinter, built the multiple-microphone recording script enabling simultaneous four-channel recording with threading, developed the initial spectrogram-based filtering approach, and served as second-semester team lead coordinating hardware-software integration and sponsor deliverables.

WebPOS — Point of Sale System

Personal Project · Solo Developer · 2026

Working at my family's restaurant that has a very slow Point of Sale (POS) system makes operations difficult and inefficient. This was the problem I was trying to solve. I wanted to create my own POS system that could take orders quickly and make menu changes efficiently. From that problem I created a full-stack web-based POS system designed for my personal use at the restaurant. This meant no more making the customer wait minutes for an order total and answers to item pricing. Runs entirely on a local network with no cloud dependency — multiple tablets, phones, and desktops can access the system simultaneously. Features menu management, real-time order tracking across devices, automated tax calculation, and role-based authentication with session management. Best thing about this is that I can change it to fit our use case.

Technology

  • HTML5
  • JSON File-Based Storage

Key Features

  • Multi-device real-time sync
  • Order lifecycle management
  • Back office menu editor

Design Decisions

  • No internet required after setup
  • No framework dependencies
  • 24-hour session timeout
  • Portable JSON data files
  • Responsive for all screens

Wi-Fi Weather Monitor

Personal Project · Solo Developer · April 2025

An ESP32-based weather station that connects to Wi-Fi, fetches real-time weather data from the OpenWeatherMap API, and displays temperature and humidity on a 1.3" SPI OLED screen. I wanted to get more hands on experience with microcontrollers and circuits so I purchased several kits and add on parts to play around with. After running through the lessons I decided that i want to work on a small project outside of the scope of the kits. The project involved debugging hardware interface mismatches (discovering the display required SPI instead of I2C), switching driver libraries, and resolving API targeting issues for accurate city-level weather data. The end product is a screen that provides up to date local weather information.

Technology

  • ESP32 Microcontroller
  • C++ / Arduino IDE
  • U8g2 Graphics Library
  • ArduinoJson
  • OpenWeatherMap API

Key Features

  • 60-second auto-refresh
  • SPI OLED display output
  • City ID-based API targeting
  • Imperial/Metric toggle
  • Breadboard prototype

Challenges Solved

  • I2C → SPI interface pivot
  • Library driver mismatch fix
  • Ambiguous city name resolution
  • OLED noise artifact diagnosis
